# しょうぎゅー！ 要件定義書

## 1. プロジェクト概要

### 1.1 目的
5歳の将棋初心者の娘と父親が、1台のiPadで一緒に将棋を楽しむためのアプリ。
子どもが直感的に操作でき、駒の動かし方を自然に覚えられる教育的配慮を備える。

### 1.2 コンセプト
- 「楽しく覚える、親子の将棋時間」
- 動物イラスト付きの駒で親しみやすく
- 補助機能で初心者でもルールを間違えずに指せる

### 1.3 ターゲットユーザー
- メイン: 5歳の将棋初心者（娘）
- サブ: 父親（対戦相手）

## 3. 機能要件（MVP詳細）

### 3.1 対局画面

#### 3.1.1 盤面
- 9x9の将棋盤を画面中央に表示
- マス目は木目調のデザイン
- 盤面の座標（筋・段）を表示（1一〜9九）

#### 3.1.2 駒
- 各駒に動物イラストを使用
- 駒と動物の対応:

| 駒 | 動物 | 理由 |
|----|------|------|
| 王将/玉将 | ライオン | 百獣の王 |
| 飛車 | 鷹（たか） | まっすぐ遠くまで飛ぶ |
| 角行 | フクロウ | 斜めに首を動かすイメージ |
| 金将 | ゾウ | 重厚で頼りになる |
| 銀将 | オオカミ | 斜めに素早く動く |
| 桂馬 | うさぎ | ぴょんと跳ねる |
| 香車 | イノシシ | まっすぐ猪突猛進 |
| 歩兵 | ひよこ | 一歩ずつ前進、成ると「ニワトリ」 |

- 成駒は動物の見た目が変化する（例: ひよこ → ニワトリ）
- 先手・後手で駒の色味を分ける（例: 青系 vs 赤系）
- 駒のサイズは子どもの指でタップしやすい大きさ（最低44x44pt）

#### 3.1.3 持ち駒エリア
- 盤面の上下にそれぞれのプレイヤーの持ち駒を表示
- 同じ種類の持ち駒は重ねて表示し、枚数をバッジで表示
- 持ち駒をタップすると打てるマスがハイライトされる

#### 3.1.4 手番表示と盤面回転
- 現在の手番を明確に表示（「あなたのばんだよ！」等のわかりやすい表現）
- 手番交代時にアニメーション付きで盤面を180度回転
- 回転中は誤操作を防ぐため入力を無効化
- 回転アニメーションは0.5〜1秒程度（子どもが認識できる速度）

### 3.2 操作補助機能

#### 3.2.1 移動可能方向の矢印表示
- 駒をタップすると、その駒が移動できる方向に矢印アイコンを表示
- 矢印は駒の種類に応じた方向パターン（前方、斜め、etc.）
- 初心者にとって「この駒はこう動く」を視覚的に学べる

#### 3.2.2 合法手ハイライト
- 駒を選択すると、実際に移動可能なマスを色付きで強調表示
- 味方の駒があるマスはハイライトしない
- 相手の駒があるマス（取れるマス）は異なる色でハイライト
- 盤外への移動や味方駒の上への移動を防止

#### 3.2.3 王手の通知
- 王手がかかった際に画面上に通知を表示
- 子ども向けのわかりやすい表現（「ライオンがあぶないよ！」等）
- 王手状態では、王手を解消する手のみハイライト

#### 3.2.4 詰み判定
- 合法手がなくなった（詰み）の判定を行う
- 勝敗決定時にお祝い/応援のアニメーション表示
- 「もういっかい あそぶ？」の再戦ボタンを表示

#### 3.2.5 待った機能
- 「もどる」ボタンで直前の手を取り消せる
- 複数手連続で戻すことが可能（MVPでは回数無制限）
- 戻した手の情報は保持し「すすむ」で再実行も可能

### 3.3 成り判定

- 成りの条件を満たした場合、選択ダイアログを表示
- 子ども向けのわかりやすい表現（「ひよこがニワトリになれるよ！ なる？」）
- 「なる」「ならない」の2択ボタンを大きく表示

### 3.4 禁手チェック

以下の禁手を自動で防止する（指せないようにする）:
- 二歩（同じ筋に歩を2枚打つ）
- 打ち歩詰め
- 行き所のない駒（相手陣最奥段の歩・香、相手陣奥から2段以内の桂）
- 王手放置（自玉が取られる手）

### 3.5 操作方式

- タップ操作のみ対応（ドラッグ&ドロップは非対応）
- 5歳児の操作を考慮し、「駒をタップ → 移動先をタップ」の2タップ方式を採用
- 誤操作しにくいシンプルな操作体系を優先する

### 3.6 将来対応が必要なルール（MVP対象外）

- 千日手（同一局面が4回出現した場合の引き分け判定）
- ステイルメイト（王手でないが合法手がない状態。将棋では極めて稀）

## 4. 非機能要件

### 4.1 対応デバイス
- iPad（第7世代以降を想定）
- 画面サイズ: 10.2インチ以上を主対象
- 横向き（ランドスケープ）表示を基本とする

### 4.2 パフォーマンス
- タッチ操作のレスポンス: 100ms以内
- 盤面回転アニメーション: 滑らかに60fps
- 合法手の計算: 体感遅延なし

### 4.3 配布方法
- PWA（Progressive Web App）として開発
- Safari からホーム画面に追加して使用
- App Store公開は不要（家族専用）
- オフラインでも動作可能（Service Worker）

### 4.4 データ保存
- 対局中の盤面状態はブラウザのローカルストレージに保存
- ページリロードや誤操作での中断に対応
- 将来の棋譜保存機能のための拡張を考慮

## 5. 技術構成

### 5.1 技術スタック

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フレームワーク | Next.js (App Router) | React ベースで PWA 対応が容易 |
| 言語 | TypeScript | 型安全性による将棋ロジックのバグ防止 |
| スタイリング | Tailwind CSS | 迅速なUI開発、レスポンシブ対応 |
| 盤面描画 | HTML/CSS + React コンポーネント | Canvas より DOM 操作が容易でアクセシビリティ確保 |
| アニメーション | Framer Motion | 盤面回転・駒移動のアニメーション |
| 状態管理 | Zustand | シンプルで軽量な状態管理 |
| PWA | next-pwa / Serwist | Service Worker によるオフライン対応 |
| テスト | Vitest | 将棋ロジック（合法手判定等）のユニットテスト |
| ホスティング | Vercel | Next.js との親和性、無料枠で十分 |

### 5.2 主要モジュール構成

```
src/
  app/                    # Next.js App Router
    page.tsx              # トップ（対局）画面
    layout.tsx            # レイアウト・PWA設定
  components/
    Board/                # 盤面コンポーネント
    Piece/                # 駒コンポーネント（動物イラスト）
    CapturedPieces/       # 持ち駒エリア
    Controls/             # 操作ボタン（待った、メニュー等）
    Dialogs/              # 成り選択・勝敗ダイアログ
    Notifications/        # 王手通知等
  lib/
    shogi/
      types.ts            # 駒・盤面の型定義
      board.ts            # 盤面の状態管理
      moves.ts            # 合法手の算出ロジック
      rules.ts            # 禁手チェック・特殊ルール
      game.ts             # ゲーム進行管理
  stores/
    gameStore.ts          # Zustand ゲーム状態ストア
  assets/
    animals/              # 動物イラスト素材
```

## 7. 開発フェーズ

### Phase 1: MVP（初回リリース）
- 将棋ロジック（駒の移動・取る・打つ・成る）
- 盤面表示と基本操作
- 動物イラスト駒
- 操作補助（矢印・ハイライト・王手通知・詰み判定）
- 待った機能
- 手番交代時の盤面回転
- PWA対応（オフライン動作・ホーム画面追加）

### Phase 2: 体験向上
- AI対戦（弱いAIから段階的に）
- 棋譜の保存・再生
- 効果音・BGM
- 待ったの回数制限モード

### Phase 3: 学習機能
- 駒の動かし方チュートリアル
- 詰将棋パズル
- 持ち時間・タイマー機能

## 8. 動物イラスト素材について

### 8.1 必要な素材一覧

| 素材 | 枚数 | 備考 |
|------|------|------|
| ライオン（王将/玉将） | 2色 | 先手・後手で色違い |
| 鷹（飛車） | 2色 + 成り | 竜王 = 金色の鷹 |
| フクロウ（角行） | 2色 + 成り | 竜馬 = 金色のフクロウ |
| ゾウ（金将） | 2色 | 成りなし |
| オオカミ（銀将） | 2色 + 成り | 成銀 = 金色のオオカミ |
| うさぎ（桂馬） | 2色 + 成り | 成桂 = 金色のうさぎ |
| イノシシ（香車） | 2色 + 成り | 成香 = 金色のイノシシ |
| ひよこ（歩兵） | 2色 | |
| ニワトリ（と金） | 2色 | ひよこの成り |

### 8.2 素材調達方針
- SVG形式で作成（拡大縮小に対応）
- シンプルでかわいらしいデザイン
- AI画像生成ツールまたはフリー素材をベースに作成
- 先手と後手は同じ動物で色味（青系/赤系）を変える
