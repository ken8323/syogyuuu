# しょうぎゅー！ 要件定義書

## 1. プロジェクト概要

### 1.1 目的
5歳の将棋初心者の娘と父親が、1台のiPadで一緒に将棋を楽しむためのアプリ。
子どもが直感的に操作でき、駒の動かし方を自然に覚えられる教育的配慮を備える。

### 1.2 コンセプト
- 「楽しく覚える、親子の将棋時間」
- 動物イラスト付きの駒で親しみやすく
- 補助機能で初心者でもルールを間違えずに指せる

### 1.3 ターゲットユーザー
- メイン: 5歳の将棋初心者（娘）
- サブ: 父親（対戦相手）

## 3. 機能要件（MVP詳細）

### 3.1 対局画面

#### 3.1.1 盤面
- 9x9の将棋盤を画面中央に表示
- マス目はCSSグラデーションによる木目テクスチャ（斜め板目模様）
- 盤面の座標（筋・段）を表示（1一〜9九）

#### 3.1.2 駒
- 各駒に動物イラストを使用
- 駒と動物の対応:

| 駒 | 動物 | 理由 |
|----|------|------|
| 王将/玉将 | ライオン | 百獣の王 |
| 飛車 | 鷹（たか） | まっすぐ遠くまで飛ぶ |
| 角行 | フクロウ | 斜めに首を動かすイメージ |
| 金将 | ゾウ | 重厚で頼りになる |
| 銀将 | オオカミ | 斜めに素早く動く |
| 桂馬 | うさぎ | ぴょんと跳ねる |
| 香車 | イノシシ | まっすぐ猪突猛進 |
| 歩兵 | ひよこ | 一歩ずつ前進、成ると「ニワトリ」 |

- 成駒は動物の見た目が変化する（例: ひよこ → ニワトリ）+ 金色の★バッジ
- 先手・後手で駒の色味を分ける（先手=青系 #3B82F6、後手=赤系 #EF4444）
- 駒の形状は五角形（CSS clip-path で将棋駒の形を再現）
- 駒の枠線はdrop-shadow（clip-path使用時はborder/ringが効かないため）
- 後手の駒は180度回転して表示（owner基準で固定。手番に依存しない）
- 駒のサイズは子どもの指でタップしやすい大きさ（最低44x44pt）

#### 3.1.3 持ち駒エリア
- 盤面の上下にそれぞれのプレイヤーの持ち駒を表示
- 同じ種類の持ち駒は重ねて表示し、枚数をバッジで表示
- 持ち駒をタップすると打てるマスがハイライトされる

#### 3.1.4 手番表示
- 現在の手番を明確に表示（「あおチームのばん」「あかチームのばん」）
- 手番表示の左端にチームカラーのアニメーションドット（scale脈動）
- 盤面は常に先手視点で固定（対面着席で1台のiPadを共有するため回転しない）
- 後手の駒は180度回転表示されるため、どちらの駒かが視覚的に区別できる

### 3.2 操作補助機能

#### 3.2.1 移動可能方向の矢印表示
- 駒をタップすると、その駒が移動できる方向に矢印アイコンを表示
- 矢印は駒の種類に応じた方向パターン（前方、斜め、etc.）
- 初心者にとって「この駒はこう動く」を視覚的に学べる

#### 3.2.2 合法手ハイライト
- 駒を選択すると、実際に移動可能なマスを色付きで強調表示
- 味方の駒があるマスはハイライトしない
- 相手の駒があるマス（取れるマス）は異なる色でハイライト
- 盤外への移動や味方駒の上への移動を防止

#### 3.2.3 王手の通知
- 王手がかかった際に画面上に通知を表示
- 子ども向けのわかりやすい表現（「ライオンがあぶないよ！」等）
- 王手状態では、王手を解消する手のみハイライト

#### 3.2.4 詰み判定
- 合法手がなくなった（詰み）の判定を行う
- 勝敗決定時にお祝い/応援のアニメーション表示
- 「もういっかい あそぶ？」の再戦ボタンを表示

#### 3.2.5 待った機能
- 「もどる」ボタンで直前の手を取り消せる
- 複数手連続で戻すことが可能（MVPでは回数無制限）
- 戻した手の情報は保持し「すすむ」で再実行も可能

#### 3.2.6 ヒント機能
- 無操作10秒（idle状態）で**レベル1**: 合法手を持つ全駒を脈動アニメーションで提示
- 無操作15秒（idle状態）で**レベル2**: おすすめの1駒とその合法手をハイライト表示
- おすすめ手の優先度: 駒を取れる手 → 前方に進める手 → 最初に見つかった手
- 手動ヒントボタン（💡）で即座にレベル2ヒントを表示
- 駒選択・手番交代・Undo/Redo・メニュー開閉でヒントをクリア

### 3.3 成り判定

- 成りの条件を満たした場合、選択ダイアログを表示
- 子ども向けのわかりやすい表現（「ひよこがニワトリになれるよ！ なる？」）
- 「なる」「ならない」の2択ボタンを大きく表示

### 3.4 禁手チェック

以下の禁手を自動で防止する（指せないようにする）:
- 二歩（同じ筋に歩を2枚打つ）
- 打ち歩詰め
- 行き所のない駒（相手陣最奥段の歩・香、相手陣奥から2段以内の桂）
- 王手放置（自玉が取られる手）

### 3.5 操作方式

- タップ操作のみ対応（ドラッグ&ドロップは非対応）
- 5歳児の操作を考慮し、「駒をタップ → 移動先をタップ」の2タップ方式を採用
- 誤操作しにくいシンプルな操作体系を優先する

### 3.6 演出機能

#### 3.6.1 駒の移動アニメーション
- 盤上移動: スプリングベースのスライド
- 桂馬: 放物線軌道アニメーション（弧を描いてぴょんと跳ねる）
- 持ち駒打ち: 目標マスでスプリングベースのポップイン
- 駒捕獲: 取られる駒がスケール縮小＋フェードアウト

→ アニメーションの具体パラメータは `screen-definition.md` セクション 2.2 駒の表示仕様 を参照

#### 3.6.2 成りアニメーション
- 通常成り: 白フラッシュ＋金色パーティクル＋★バッジのポップイン
- 強制成り: 短縮版（フラッシュ・パーティクルのみ）

→ タイミング・パラメータの詳細は `screen-definition.md` セクション 2.2 を参照

#### 3.6.3 駒のアイドルアニメーション
- 各動物に固有のアイドルアニメーション（Framer Motion variants）
- 例: 鷹の羽ばたき、ゾウの体揺らし、うさぎの耳ぴくぴく等
- 選択中の駒は上下バウンスアニメーション

#### 3.6.4 ほめメッセージ
- 画面中央にオーバーレイ表示、一定時間後に自動消去
- トリガー: 駒取得（初回/通常）、成り、10手ごとのマイルストーン
- 直前と同じメッセージは連続しない

→ 表示仕様は `screen-definition.md` セクション 2.8 を参照
→ メッセージ一覧・選出ロジックは `data-type-definition.md` セクション 11.8 を参照
→ トリガー条件の詳細は `state-transition.md` セクション 2.6 を参照

#### 3.6.5 効果音
- Web Audio API によるリアルタイム合成音（音声ファイル不使用）
- AudioContext は最初のユーザー操作後に遅延初期化（iOS Safari制限への対応）
- 操作種別ごとに異なる効果音を再生（選択・配置・捕獲・成り・王手・勝利等）

→ SoundId 一覧・周波数・波形の詳細は `data-type-definition.md` セクション 11.6 を参照
→ 各操作での発火タイミングは `state-transition.md` セクション 2.1 を参照

#### 3.6.6 触覚フィードバック
- Web Vibration API を使用（未対応環境ではスキップ）
- ミュート設定と連動（isMuted時は無効化）
- 操作種別ごとに異なる振動パターン

→ 振動パターンの詳細は `data-type-definition.md` セクション 11.7 を参照

### 3.7 将来対応が必要なルール（MVP対象外）

- 千日手（同一局面が4回出現した場合の引き分け判定）
- ステイルメイト（王手でないが合法手がない状態。将棋では極めて稀）

## 4. 非機能要件

### 4.1 対応デバイス
- iPad（第7世代以降を想定）
- 画面サイズ: 10.2インチ以上を主対象
- 横向き（ランドスケープ）表示を基本とする

### 4.2 パフォーマンス
- タッチ操作のレスポンス: 100ms以内
- UIアニメーション: 滑らかに60fps
- 合法手の計算: 体感遅延なし

### 4.3 配布方法
- PWA（Progressive Web App）として開発
- Safari からホーム画面に追加して使用
- App Store公開は不要（家族専用）
- オフラインでも動作可能（Service Worker）

### 4.4 データ保存
- 対局中の盤面状態はブラウザのローカルストレージに保存
- ページリロードや誤操作での中断に対応
- ミュート設定もローカルストレージに保存（対局をまたいで永続化）
- 将来の棋譜保存機能のための拡張を考慮

## 8. 動物イラスト素材について

### 8.1 必要な素材一覧

| 素材 | 枚数 | 備考 |
|------|------|------|
| ライオン（王将/玉将） | 2色 | 先手・後手で色違い |
| 鷹（飛車） | 2色 + 成り | 竜王 = 金色の鷹 |
| フクロウ（角行） | 2色 + 成り | 竜馬 = 金色のフクロウ |
| ゾウ（金将） | 2色 | 成りなし |
| オオカミ（銀将） | 2色 + 成り | 成銀 = 金色のオオカミ |
| うさぎ（桂馬） | 2色 + 成り | 成桂 = 金色のうさぎ |
| イノシシ（香車） | 2色 + 成り | 成香 = 金色のイノシシ |
| ひよこ（歩兵） | 2色 | |
| ニワトリ（と金） | 2色 | ひよこの成り |

### 8.2 素材実装方針
- React SVGコンポーネントとして実装（`src/components/Piece/animals.tsx`）
- シンプルでかわいらしいデザイン（ピンクの頬紅、大きな目、丸みのある形状）
- 先手と後手は同じ動物で色味を変える（props で primary/dark カラーを受け取る）
- 成駒は金色の★バッジを右上に表示（`isPromoted` prop で制御）
