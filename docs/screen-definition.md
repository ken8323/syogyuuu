## 2. 画面詳細

### 2.1 SCR-001: タイトル画面

#### 目的
アプリ起動時に表示。対局を開始する導線を提供する。

#### レイアウト（ランドスケープ / 1366x1024 基準）

```
+----------------------------------------------------------+
|                                                          |
|                                                          |
|               [動物たちのイラスト]                         |
|                                                          |
|              しょうぎゅー！                                |
|              （アプリタイトル）                             |
|                                                          |
|           +----------------------------+                 |
|           |    あそぶ！                  |                 |
|           +----------------------------+                 |
|                                                          |
|           +----------------------------+                 |
|           |    つづきから                 |                 |
|           +----------------------------+                 |
|                                                          |
+----------------------------------------------------------+
```

#### コンポーネント一覧

| ID | コンポーネント | 説明 |
|----|--------------|------|
| T-001 | タイトルロゴ | アプリ名「しょうぎゅー！」をかわいいフォントで表示 |
| T-002 | キービジュアル | 動物駒たちが並んでいるイラスト |
| T-003 | 「あそぶ！」ボタン | 新規対局を開始。タップで SCR-002 へ遷移 |
| T-004 | 「つづきから」ボタン | 中断した対局を再開。保存データがない場合は非表示 |

#### インタラクション

| 操作 | 動作 |
|------|------|
| 「あそぶ！」タップ | 初期配置で SCR-002（対局画面）へ遷移 |
| 「つづきから」タップ | ローカルストレージから盤面を復元し SCR-002 へ遷移 |

#### 駒の表示仕様

| 状態 | 表示 |
|------|------|
| 通常（未選択） | 動物イラスト + 駒名（ひらがな）。先手=青系、後手=赤系 |
| 選択中 | 駒の周囲に光彩エフェクト + 上下にバウンスアニメーション |
| 成駒 | 成った動物イラスト（例: ニワトリ）+ 金色の枠 |

#### マスの表示仕様

| 状態 | 色/表示 |
|------|--------|
| 通常 | 木目調の背景 |
| 合法手（移動可能） | 緑色の半透明円（中央にドット） |
| 合法手（駒が取れる） | 赤色の半透明円 |
| 直前の手（移動元） | 黄色の薄い背景 |
| 直前の手（移動先） | 黄色のやや濃い背景 |
| 選択中の駒がいるマス | 水色の背景 |

#### インタラクション詳細

| 操作 | 条件 | 動作 |
|------|------|------|
| 自分の駒をタップ | 自分の手番 | 駒を選択状態にする。合法手ハイライト + 矢印表示 |
| 選択中に合法手マスをタップ | 駒選択中 | 駒を移動。成り条件なら SCR-003 表示。手番交代 |
| 選択中に合法手外をタップ | 駒選択中 | 選択解除 |
| 選択中に別の自分の駒をタップ | 駒選択中 | 選択を切り替え |
| 相手の駒をタップ | 自分の手番 | 何も起きない（反応なし） |
| 持ち駒をタップ | 自分の手番 | 打てるマスをハイライト表示 |
| 持ち駒選択中にマスをタップ | 打てるマス | 駒を打つ。手番交代 |
| 「もどる」タップ | 手の履歴あり | 直前の1手を取り消す |
| 「すすむ」タップ | 戻した手あり | 取り消した手を再実行 |
| 「メニュー」タップ | 常時 | SCR-007 を表示 |

#### 盤面回転の仕様

**実装方式: 論理座標変換方式（CSS rotate は使用しない）**

CSSのrotateで盤面全体を回転させると、座標ラベルの文字反転や持ち駒エリアの意図しない回転が発生するため、
**描画時の座標変換**で「回転したように見せる」方式を採用する。

| 項目 | 仕様 |
|------|------|
| トリガー | 手番交代時（駒を動かした直後） |
| 実装方式 | 盤面の描画座標を反転して再描画（row, col の描画順を逆にする） |
| アニメーション | Framer Motion で盤面コンテナを fade-out → 座標反転 → fade-in |
| アニメーション時間 | 0.8秒 |
| イージング | ease-in-out |
| 回転中の操作 | 全タッチイベントを無効化（pointer-events: none） |
| 回転後 | 手番側の陣地が画面下側に来る |
| 座標ラベル | 描画順に連動して正しい向きで常に表示される（文字は回転しない） |
| 持ち駒エリア | 回転対象外。常に「上=相手、下=自分」の配置を維持 |

**座標変換ルール:**
- 先手の手番時: `displayRow = row`, `displayCol = col`（そのまま描画）
- 後手の手番時: `displayRow = 8 - row`, `displayCol = 8 - col`（反転して描画）
- 内部ロジック（合法手計算等）は常に同じ座標系で動作し、表示時のみ変換する

### 2.3 SCR-003: 成りダイアログ

#### 目的
成りの条件を満たした駒に対し、成るかどうかを選択させる。

#### レイアウト

```
+----------------------------------------------------------+
|                  （背景を暗くする）                         |
|                                                          |
|          +--------------------------------+              |
|          |                                |              |
|          |   [ひよこ] → [ニワトリ]          |              |
|          |                                |              |
|          |   ひよこがニワトリに               |              |
|          |   なれるよ！ なる？               |              |
|          |                                |              |
|          |  +-----------+ +-----------+   |              |
|          |  |   なる！   | | ならない  |   |              |
|          |  +-----------+ +-----------+   |              |
|          |                                |              |
|          +--------------------------------+              |
|                                                          |
+----------------------------------------------------------+
```

#### コンポーネント一覧

| ID | コンポーネント | 説明 |
|----|--------------|------|
| P-001 | Overlay | 半透明黒の背景オーバーレイ |
| P-002 | PromotionDialog | 白背景の角丸ダイアログ |
| P-003 | BeforeImage | 成り前の動物イラスト |
| P-004 | ArrowIcon | → の変身アイコン |
| P-005 | AfterImage | 成り後の動物イラスト |
| P-006 | MessageText | 「○○が△△になれるよ！ なる？」のテキスト |
| P-007 | PromoteButton | 「なる！」ボタン（大きめ・目立つ色） |
| P-008 | DeclineButton | 「ならない」ボタン（控えめ色） |

#### メッセージ対応表

| 駒 | メッセージ |
|----|----------|
| 歩兵 | 「ひよこがニワトリになれるよ！ なる？」 |
| 香車 | 「イノシシがパワーアップできるよ！ なる？」 |
| 桂馬 | 「うさぎがパワーアップできるよ！ なる？」 |
| 銀将 | 「オオカミがパワーアップできるよ！ なる？」 |
| 角行 | 「フクロウがパワーアップできるよ！ なる？」 |
| 飛車 | 「たかがパワーアップできるよ！ なる？」 |

#### インタラクション

| 操作 | 動作 |
|------|------|
| 「なる！」タップ | 駒を成駒に変更。手番交代 → SCR-006 |
| 「ならない」タップ | 駒はそのまま。手番交代 → SCR-006 |
| オーバーレイタップ | 何も起きない（必ずどちらかを選択させる） |

#### 強制成りの場合
行き所のない位置（相手陣最奥段の歩・香、相手陣奥から2段以内の桂）に進む場合:
- ダイアログを表示せず自動的に成る
- トースト通知を0.8秒表示（例: 「ひよこがニワトリになったよ！」「うさぎがパワーアップしたよ！」）

### 2.6 SCR-006: 手番交代画面

#### 目的
盤面回転中の画面。誤操作防止と「次は相手の番」の意識づけ。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 表示タイミング | 駒を動かした直後〜座標反転アニメーション完了まで |
| 交代中の表示 | 盤面の fade-out → 座標反転 → fade-in アニメーション |
| タッチ操作 | 全無効（pointer-events: none） |
| 所要時間 | 0.8秒 |
| 完了後 | 手番表示を更新し、操作を再開 |

※ 独立した画面ではなく、SCR-002 上のアニメーション状態として実装する。
※ CSS rotate ではなく論理座標変換方式を使用（SCR-002 の盤面回転仕様を参照）。

## 4. レスポンシブ対応

### 4.1 基準解像度

| デバイス | 解像度（論理ピクセル） | 対応 |
|---------|---------------------|------|
| iPad (10.2") | 1080 x 810 | 主対象 |
| iPad Air (10.9") | 1180 x 820 | 対応 |
| iPad Pro 11" | 1194 x 834 | 対応 |
| iPad Pro 12.9" | 1366 x 1024 | 対応 |

### 4.2 サイズ計算方針

| 要素 | サイズ計算 |
|------|----------|
| 盤面 | 画面高さの65%を最大として正方形で表示 |
| マス目 | 盤面サイズ / 9 |
| 駒 | マス目サイズの90% |
| 持ち駒エリア | 画面高さの12%（上下各）。最大7種の駒を横並びで44pt以上確保 |
| 操作バー | 画面高さの8% |
| 最小タッチサイズ | 44 x 44 pt（Apple HIG 準拠） |

### 4.3 持ち駒エリアのレイアウト補足

iPad 10.2" (810pt高) の場合:
- 持ち駒エリア高さ: 810 x 12% = 約97pt（44pt以上を確保可能）
- 横幅: 盤面幅と同幅。7種の駒を均等配置（各駒間にバッジ表示余白を含む）
- 持ち駒が0枚の種類は薄くグレーアウトして表示位置を固定する
