## 2. 画面詳細

### 2.1 SCR-001: タイトル画面

#### 目的
アプリ起動時に表示。対局を開始する導線を提供する。

#### レイアウト（ランドスケープ / 1366x1024 基準）

```
+----------------------------------------------------------+
|                                                          |
|                                                          |
|               [動物たちのイラスト]                         |
|                                                          |
|              しょうぎゅー！                                |
|              （アプリタイトル）                             |
|                                                          |
|           +----------------------------+                 |
|           |    あそぶ！                  |                 |
|           +----------------------------+                 |
|                                                          |
|           +----------------------------+                 |
|           |    つづきから                 |                 |
|           +----------------------------+                 |
|                                                          |
+----------------------------------------------------------+
```

#### コンポーネント一覧

| ID | コンポーネント | 説明 |
|----|--------------|------|
| T-001 | タイトルロゴ | アプリ名「しょうぎゅー！」をZen Maru Gothicで表示 |
| T-002 | キービジュアル | 動物駒たちが並んでいるイラスト |
| T-003 | 「あそぶ！」ボタン | 新規対局を開始。タップで SCR-002 へ遷移 |
| T-004 | 「つづきから」ボタン | 中断した対局を再開。保存データがない場合は非表示 |
| T-005 | バージョン番号 | 画面右下に `v{version}` を小さく表示。`package.json` の `version` フィールドをビルド時に埋め込む（`NEXT_PUBLIC_APP_VERSION`）。amber-400/60 の薄い色で子どもの視線を邪魔しない |
| T-006 | 「こまのうごきかた」ボタン | 駒の動き方ヘルプ画面（SCR-010）を開く |

#### 保存データの判定
- `moveHistory.moves.length > 0` の場合に「つづきから」ボタンを表示

#### インタラクション

| 操作 | 動作 |
|------|------|
| 「あそぶ！」タップ | 初期配置で SCR-002（対局画面）へ遷移 |
| 「つづきから」タップ | ローカルストレージから盤面を復元し SCR-002 へ遷移 |

#### 駒の表示仕様

| 状態 | 表示 |
|------|------|
| 通常（未選択） | 五角形clip-path + 動物PNG画像（`public/icons/` 配下）+ 駒名（ひらがな）。後手の駒は180度回転（owner基準固定）。各動物固有のアイドルアニメーション |
| 選択中 | drop-shadow光彩エフェクト + 上下にバウンスアニメーション（y: 0→-4→0, 0.5s） |
| 成駒 | 成った動物のPNG画像（成駒専用画像）+ 金色のdrop-shadow枠 |
| ヒント対象（level 1） | 脈動アニメーション（scale / opacity の周期変化） |
| ヒント対象（level 2） | 特定の1駒が脈動 + その合法手マスがハイライト |
| 移動アニメーション中 | AnimatingPiece コンポーネントで描画。盤面上の駒は非表示 |
| 成りアニメーション中 | PromotionEffect コンポーネントが重ねて描画 |

#### 駒カラー（和風くすみカラー）

| 要素 | 先手（藍系） | 後手（茜系） |
|------|------------|------------|
| 背景色 | `#C8D8E4`（薄墨藍） | `#E4C8C8`（薄茜色） |
| 枠線 drop-shadow | `#7A9CB8` | `#B87A7A` |
| テキスト色 | `#2C4A5E` | `#5E2C2C` |
| 持ち駒エリア ring | `#7A9CB8` | `#B87A7A` |

#### 盤面の質感仕様

##### 木目テクスチャ（マス背景）
- CSSグラデーションで板目を表現（画像不使用）
- 5層構造:
  1. 太い木目筋: 105度斜めグラデーション（`rgba(139,90,43,0.10)`、幅3%）
  2. 中間の木目筋: 105度斜めグラデーション（`rgba(139,90,43,0.07)`、幅2%）
  3. 細い木目筋: 105度斜めグラデーション（`rgba(139,90,43,0.04)`、幅1%）
  4. 年輪風の微細パターン: 繰り返しグラデーション（`rgba(160,120,60,0.03)`、4px間隔）
  5. 上部ハイライト: 縦グラデーション（`rgba(255,220,150,0.25)`、上明→下暗）
- ベース色: `#d4a843` → `#c49132` の斜めグラデーション

##### 漆風縁取り（盤面外枠）
- 外枠: 3px の濃い焦げ茶（`#3d1c00`）
- 内側に漆の光沢を表現する box-shadow:
  - ハイライト: `inset 2px 2px 4px rgba(255,180,60,0.4)`（左上の光沢）
  - シャドウ: `inset -2px -2px 4px rgba(0,0,0,0.5)`（右下の影）
  - 漆の深み: `inset 0 0 8px rgba(80,30,0,0.3)`（全体の深み）
- 外側ドロップシャドウ: `0 4px 16px rgba(0,0,0,0.35)`

##### 罫線（マス目の溝）
- border の代わりに box-shadow で木彫り風の溝を表現
- 右辺の溝: `1px 0 0 rgba(100,60,20,0.4)` + `-1px 0 0 rgba(255,210,130,0.15)`（影+ハイライト）
- 下辺の溝: `0 1px 0 rgba(100,60,20,0.4)` + `0 -1px 0 rgba(255,210,130,0.15)`（影+ハイライト）

#### マスの表示仕様

| 状態 | 色/表示 |
|------|--------|
| 通常 | CSSグラデーションによる木目テクスチャ（上記「盤面の質感仕様」参照） |
| 合法手（移動可能） | 緑色の半透明円（中央にドット） |
| 合法手（駒が取れる） | 赤色の半透明円 |
| 直前の手（移動元） | 黄色の薄い背景 |
| 直前の手（移動先） | 黄色のやや濃い背景 |
| 選択中の駒がいるマス | 水色の背景 |
| ヒント合法手マス | ヒント用のハイライト |
| 王手中の王のマス | 赤系の強調背景 |

#### インタラクション詳細

| 操作 | 条件 | 動作 |
|------|------|------|
| 自分の駒をタップ | 自分の手番（idle / piece_selected） | 駒を選択状態にする。合法手ハイライト + 矢印表示 |
| 選択中に合法手マスをタップ | 駒選択中 | 移動アニメーション開始。成り条件なら SCR-003 表示。手番交代 |
| 選択中に合法手外をタップ | 駒選択中 | 選択解除 |
| 選択中の駒を再タップ | 駒選択中（piece_selected） | 選択解除。ハイライト・矢印消去 |
| 選択中に別の自分の駒をタップ | 駒選択中 | 選択を切り替え |
| 相手の駒をタップ | 自分の手番 | 何も起きない（反応なし） |
| 持ち駒をタップ | 自分の手番（idle / captured_selected） | 打てるマスをハイライト表示 |
| 選択中の持ち駒を再タップ | 持ち駒選択中 | 選択解除 |
| 持ち駒選択中にマスをタップ | 打てるマス | 駒打ちアニメーション開始。手番交代 |
| 「もどる」タップ | 手の履歴あり & idle | 直前の1手を取り消す。ヒントクリア |
| 「すすむ」タップ | 戻した手あり & idle | 取り消した手を再実行。ヒントクリア |
| 💡 タップ | idle & メニュー閉 | おすすめ手ヒント（レベル2）を即座に表示 |
| 🔊/🔇 タップ | 常時 | 効果音＋触覚のミュート切替 |
| 「☰」タップ | 常時 | SCR-007 を表示 |

#### 盤面表示の仕様

**実装方式: 先手視点固定（回転なし）**

1台のiPadを対面で共有するため、盤面は常に先手視点で固定表示する。
後手の駒は180度回転表示されるため、どちらの駒かが視覚的に区別できる。

| 項目 | 仕様 |
|------|------|
| 描画方式 | 常に先手視点で固定。`displayRow = row`, `displayCol = col` |
| 座標ラベル | 筋: 9〜1（左→右）、段: 一〜九（上→下）で固定 |
| 持ち駒エリア | 上=後手（gote）、下=先手（sente）で固定 |
| 後手の駒 | `isOpponent={piece.owner === 'gote'}` で180度回転表示 |
| 手番交代 | 手番表示テキストの更新のみ。盤面の見た目は変化しない |

#### 操作バーの手番表示カラー

| 手番 | ControlBar左端ボーダー | 手番テキスト背景 | ドット色 |
|------|----------------------|-----------------|---------|
| 先手（あおチーム） | #3B82F6 | bg-blue-100 text-blue-900 | bg-blue-500 |
| 後手（あかチーム） | #EF4444 | bg-red-100 text-red-900 | bg-red-500 |

### 2.3 SCR-003: 成りダイアログ

#### 目的
成りの条件を満たした駒に対し、成るかどうかを選択させる。

#### レイアウト

```
+----------------------------------------------------------+
|                  （背景を暗くする）                         |
|                                                          |
|          +--------------------------------+              |
|          |                                |              |
|          |   [ひよこ] → [ニワトリ]          |              |
|          |                                |              |
|          |   ひよこがニワトリに               |              |
|          |   なれるよ！ なる？               |              |
|          |                                |              |
|          |  +-----------+ +-----------+   |              |
|          |  |   なる！   | | ならない  |   |              |
|          |  +-----------+ +-----------+   |              |
|          |                                |              |
|          +--------------------------------+              |
|                                                          |
+----------------------------------------------------------+
```

#### コンポーネント一覧

| ID | コンポーネント | 説明 |
|----|--------------|------|
| P-001 | Overlay | 半透明黒の背景オーバーレイ |
| P-002 | PromotionDialog | 白背景の角丸ダイアログ |
| P-003 | BeforeImage | 成り前の動物イラスト |
| P-004 | ArrowIcon | → の変身アイコン |
| P-005 | AfterImage | 成り後の動物イラスト |
| P-006 | MessageText | 「○○が△△になれるよ！ なる？」のテキスト |
| P-007 | PromoteButton | 「なる！」ボタン（大きめ・目立つ色） |
| P-008 | DeclineButton | 「ならない」ボタン（控えめ色） |

#### メッセージ対応表

| 駒 | メッセージ |
|----|----------|
| 歩兵 | 「ひよこがニワトリになれるよ！ なる？」 |
| 香車 | 「イノシシがパワーアップできるよ！ なる？」 |
| 桂馬 | 「うさぎがパワーアップできるよ！ なる？」 |
| 銀将 | 「オオカミがパワーアップできるよ！ なる？」 |
| 角行 | 「フクロウがパワーアップできるよ！ なる？」 |
| 飛車 | 「たかがパワーアップできるよ！ なる？」 |

#### 後手の表示

- 後手（gote）の場合、ダイアログ全体（アイコン・メッセージ・ボタン）を `rotate(180deg)` で回転表示する
- 向かい合って操作するため、後手側から読める向きにする

#### インタラクション

| 操作 | 動作 | 効果音 | 触覚 |
|------|------|--------|------|
| 「なる！」タップ | 成りアニメーション再生 → 駒を成駒に変更 → 手番交代 | promote | hapticPromote |
| 「ならない」タップ | 駒はそのまま → 手番交代 | なし | なし |
| オーバーレイタップ | 何も起きない（必ずどちらかを選択させる） | - | - |

#### 強制成りの場合
行き所のない位置（相手陣最奥段の歩・香、相手陣奥から2段以内の桂）に進む場合:
- ダイアログを表示せず自動的に成る
- 短縮版成りアニメーション（0.5秒）を再生
- アニメーション後にForcedPromotionToastを表示（例: 「ひよこがニワトリになったよ！」「うさぎがパワーアップしたよ！」）

### 2.4 SCR-004: 王手通知

#### 目的
王手がかかったことを、わかりやすく子どもに伝える。

#### レイアウト

```
+----------------------------------------------------------+
|                                                          |
|         +------------------------------+                 |
|         |  [ライオンアイコン]            |                 |
|         |  ライオンがあぶないよ！         |                 |
|         +------------------------------+                 |
|                                                          |
|                    （盤面はそのまま）                       |
+----------------------------------------------------------+
```

#### コンポーネント一覧

| ID | コンポーネント | 説明 |
|----|--------------|------|
| C-001 | CheckBanner | 画面上部に表示されるバナー |
| C-002 | KingIcon | 危険な状態のライオンアイコン（汗マーク付きなど） |
| C-003 | CheckMessage | 「ライオンがあぶないよ！」テキスト |

#### 表示仕様

| 項目 | 仕様 |
|------|------|
| 表示タイミング | 手番交代完了後、王手が検出された場合 |
| アニメーション | 上からスライドイン → 1.5秒表示 → フェードアウト |
| 操作への影響 | バナー表示中も盤面操作は可能（completeCheckNotify後に idle 遷移） |
| 盤面の制限 | 王手解消手のみ合法手としてハイライト |
| 効果音 | check（220Hz+277Hz不協和音、0.35秒） |
| 触覚 | hapticCheck: [30, 50, 30, 50, 30]ms |

### 2.5 SCR-005: 勝敗ダイアログ

#### 目的
詰み発生時に勝敗を表示し、再戦を促す。

#### レイアウト

```
+----------------------------------------------------------+
|                  （背景を暗くする）                         |
|                                                          |
|          +--------------------------------+              |
|          |                                |              |
|          |     [お祝いアニメーション]        |              |
|          |                                |              |
|          |       つみ！                    |              |
|          |    あかチームのかち！             |              |
```

#### パズルフロー

1. 初期盤面を表示（攻め方=先手の手番）

2. プレイヤーが1手目を指す
3. → 正解手でなければ不正解（PZ-006 → リセット）
4. → 正解手なら、相手（玉方）が最善応手を自動で指す（0.5秒ディレイ）
5. プレイヤーが3手目を指す
6. → 詰みなら正解（PZ-005 表示）
7. → 詰みでなければ不正解（PZ-006 → リセット）
