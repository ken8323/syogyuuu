## 2. 画面詳細

### 2.1 SCR-001: タイトル画面

#### 目的
アプリ起動時に表示。対局を開始する導線を提供する。

#### レイアウト（ランドスケープ / 1366x1024 基準）

```
+----------------------------------------------------------+
|                                                          |
|                                                          |
|               [動物たちのイラスト]                         |
|                                                          |
|              しょうぎゅー！                                |
|              （アプリタイトル）                             |
|                                                          |
|           +----------------------------+                 |
|           |    あそぶ！                  |                 |
|           +----------------------------+                 |
|                                                          |
|           +----------------------------+                 |
|           |    つづきから                 |                 |
|           +----------------------------+                 |
|                                                          |
+----------------------------------------------------------+
```

#### コンポーネント一覧

| ID | コンポーネント | 説明 |
|----|--------------|------|
| T-001 | タイトルロゴ | アプリ名「しょうぎゅー！」をZen Maru Gothicで表示 |
| T-002 | キービジュアル | 動物駒たちが並んでいるイラスト |
| T-003 | 「あそぶ！」ボタン | 新規対局を開始。タップで SCR-002 へ遷移 |
| T-004 | 「つづきから」ボタン | 中断した対局を再開。保存データがない場合は非表示 |

#### インタラクション

| 操作 | 動作 |
|------|------|
| 「あそぶ！」タップ | 初期配置で SCR-002（対局画面）へ遷移 |
| 「つづきから」タップ | ローカルストレージから盤面を復元し SCR-002 へ遷移 |

| G-006 | CaptureHighlight | マス上 | 取れる駒のマスの赤系ハイライト |
| G-007 | LastMoveHighlight | マス上 | 直前の手の移動元・移動先を薄くハイライト |
| G-008 | CapturedPiecesArea（上） | 盤面上部 | 相手側の持ち駒表示エリア |
| G-009 | CapturedPiecesArea（下） | 盤面下部 | 手番側の持ち駒表示エリア |
| G-010 | CapturedPiece | 持ち駒エリア内 | 個別の持ち駒。枚数バッジ付き |
| G-011 | TurnIndicator | 下部バー中央 | 「あなたのばんだよ！」の手番表示 |
| G-012 | UndoButton | 下部バー左 | 「もどる」ボタン |
| G-013 | RedoButton | 下部バー右寄り | 「すすむ」ボタン |
| G-014 | MenuButton | 下部バー右端 | メニューを開くボタン |
| G-015 | ColumnLabels | 盤面上端 | 筋ラベル（9〜1） |
| G-016 | RowLabels | 盤面右端 | 段ラベル（一〜九） |

#### 駒の表示仕様

| 状態 | 表示 |
|------|------|
| 通常（未選択） | 五角形clip-path + 動物SVG + 駒名（ひらがな）。先手=青系、後手=赤系。後手の駒は180度回転（owner基準固定） |
| 選択中 | drop-shadow光彩エフェクト + 上下にバウンスアニメーション（y: 0→-4→0, 0.5s） |
| 成駒 | 成った動物SVG + 金色の★バッジ（右上）+ 金色のdrop-shadow枠 |

#### マスの表示仕様

| 状態 | 色/表示 |
|------|--------|
| 通常 | CSSグラデーションによる木目テクスチャ（斜め板目模様） |
| 合法手（移動可能） | 緑色の半透明円（中央にドット） |
| 合法手（駒が取れる） | 赤色の半透明円 |
| 直前の手（移動元） | 黄色の薄い背景 |
| 直前の手（移動先） | 黄色のやや濃い背景 |
| 選択中の駒がいるマス | 水色の背景 |

#### インタラクション詳細

| 操作 | 条件 | 動作 |
|------|------|------|
| 自分の駒をタップ | 自分の手番 | 駒を選択状態にする。合法手ハイライト + 矢印表示 |
| 選択中に合法手マスをタップ | 駒選択中 | 駒を移動。成り条件なら SCR-003 表示。手番交代 |
| 選択中に合法手外をタップ | 駒選択中 | 選択解除 |
| 選択中に別の自分の駒をタップ | 駒選択中 | 選択を切り替え |
| 相手の駒をタップ | 自分の手番 | 何も起きない（反応なし） |
| 持ち駒をタップ | 自分の手番 | 打てるマスをハイライト表示 |
| 持ち駒選択中にマスをタップ | 打てるマス | 駒を打つ。手番交代 |
| 「もどる」タップ | 手の履歴あり | 直前の1手を取り消す |
| 「すすむ」タップ | 戻した手あり | 取り消した手を再実行 |
| 「メニュー」タップ | 常時 | SCR-007 を表示 |

#### 盤面表示の仕様

**実装方式: 先手視点固定（回転なし）**

1台のiPadを対面で共有するため、盤面は常に先手視点で固定表示する。
後手の駒は180度回転表示されるため、どちらの駒かが視覚的に区別できる。

| 項目 | 仕様 |
|------|------|
| 描画方式 | 常に先手視点で固定。`displayRow = row`, `displayCol = col` |
| 座標ラベル | 筋: 9〜1（左→右）、段: 一〜九（上→下）で固定 |
| 持ち駒エリア | 上=後手（gote）、下=先手（sente）で固定 |
| 後手の駒 | `isOpponent={piece.owner === 'gote'}` で180度回転表示 |
| 手番交代 | 手番表示テキストの更新のみ。盤面の見た目は変化しない |

### 2.3 SCR-003: 成りダイアログ

#### 目的
成りの条件を満たした駒に対し、成るかどうかを選択させる。

#### レイアウト

```
+----------------------------------------------------------+
|                  （背景を暗くする）                         |
|                                                          |
|          +--------------------------------+              |
|          |                                |              |
|          |   [ひよこ] → [ニワトリ]          |              |
|          |                                |              |
|          |   ひよこがニワトリに               |              |
|          |   なれるよ！ なる？               |              |
|          |                                |              |
|          |  +-----------+ +-----------+   |              |
|          |  |   なる！   | | ならない  |   |              |
|          |  +-----------+ +-----------+   |              |
|          |                                |              |
|          +--------------------------------+              |
|                                                          |
+----------------------------------------------------------+
```

#### コンポーネント一覧

| ID | コンポーネント | 説明 |
|----|--------------|------|
| P-001 | Overlay | 半透明黒の背景オーバーレイ |
| P-002 | PromotionDialog | 白背景の角丸ダイアログ |
| P-003 | BeforeImage | 成り前の動物イラスト |
| P-004 | ArrowIcon | → の変身アイコン |
| P-005 | AfterImage | 成り後の動物イラスト |
| P-006 | MessageText | 「○○が△△になれるよ！ なる？」のテキスト |
| P-007 | PromoteButton | 「なる！」ボタン（大きめ・目立つ色） |
| P-008 | DeclineButton | 「ならない」ボタン（控えめ色） |

#### メッセージ対応表

| 駒 | メッセージ |
|----|----------|
| 歩兵 | 「ひよこがニワトリになれるよ！ なる？」 |
| 香車 | 「イノシシがパワーアップできるよ！ なる？」 |
| 桂馬 | 「うさぎがパワーアップできるよ！ なる？」 |
| 銀将 | 「オオカミがパワーアップできるよ！ なる？」 |
| 角行 | 「フクロウがパワーアップできるよ！ なる？」 |
| 飛車 | 「たかがパワーアップできるよ！ なる？」 |

#### インタラクション

| 操作 | 動作 |
|------|------|
| 「なる！」タップ | 駒を成駒に変更。手番交代 |
| 「ならない」タップ | 駒はそのまま。手番交代 |
| オーバーレイタップ | 何も起きない（必ずどちらかを選択させる） |

#### 強制成りの場合
行き所のない位置（相手陣最奥段の歩・香、相手陣奥から2段以内の桂）に進む場合:
- ダイアログを表示せず自動的に成る
- トースト通知を0.8秒表示（例: 「ひよこがニワトリになったよ！」「うさぎがパワーアップしたよ！」）

### 2.6 SCR-006: 手番交代

#### 目的
手番を交代し、次のプレイヤーの入力待ちに遷移する。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 実装方式 | GamePhase の `turn_switching` 状態を経由して即座に手番交代 |
| 盤面の変化 | なし（盤面は常に先手視点で固定） |
| 手番表示 | ControlBar の表示テキストを更新 |
| 完了後 | 王手判定 → CHECK_NOTIFY or IDLE に遷移 |

※ 独立した画面ではなく、SCR-002 上の状態遷移として実装する。

## 4. レスポンシブ対応

### 4.1 基準解像度

| デバイス | 解像度（論理ピクセル） | 対応 |
|---------|---------------------|------|
| iPad (10.2") | 1080 x 810 | 主対象 |
| iPad Air (10.9") | 1180 x 820 | 対応 |
| iPad Pro 11" | 1194 x 834 | 対応 |
| iPad Pro 12.9" | 1366 x 1024 | 対応 |

### 4.2 サイズ計算方針

| 要素 | サイズ計算 |
|------|----------|
| 盤面 | 画面高さの65%を最大として正方形で表示 |
| マス目 | 盤面サイズ / 9 |
| 駒 | マス目サイズの90% |
| 持ち駒エリア | 画面高さの12%（上下各）。最大7種の駒を横並びで44pt以上確保 |
| 操作バー | 画面高さの8% |
| 最小タッチサイズ | 44 x 44 pt（Apple HIG 準拠） |

### 4.3 持ち駒エリアのレイアウト補足

iPad 10.2" (810pt高) の場合:
- 持ち駒エリア高さ: 810 x 12% = 約97pt（44pt以上を確保可能）
- 横幅: 盤面幅と同幅。7種の駒を均等配置（各駒間にバッジ表示余白を含む）
- 持ち駒が0枚の種類は薄くグレーアウトして表示位置を固定する
