# retrospective

CI失敗・レビュー差し戻し・本番バグ発生時の振り返りスキル。

---
argument-hint: "<何が起きたかの簡潔な説明>"
---

## 手順

1. 事象を整理する
   - 何が起きたか（事象）
   - なぜ起きたか（原因）
   - どう防ぐか（対策）

2. `docs/learnings.md` に以下のフォーマットで追記する

```markdown
### YYYY-MM-DD: <タイトル>

- **事象**: <何が起きたか>
- **原因**: <なぜ起きたか>
- **対策**: <どう防ぐか>
- **関連Issue**: #<Issue番号>
```

3. GitHub Issue を作成する
   - タイトル: `[振り返り] <事象の要約>`
   - ラベル: `retrospective`
   - 本文: 上記の事象・原因・対策

4. 作成した振り返りIssueをクローズする（`docs/learnings.md` に記録済みのため、対応不要の記録用Issueを残さない）

5. CLAUDE.md にルールを追加する場合
   - 既存ルールとの重複・矛盾がないか確認する
   - 追加する場合は適切なセクションに配置する

## 注意事項

- 修正完了後に必ず実行すること（修正完了 ≠ 対応完了）
- 表面的な原因だけでなく、根本原因を掘り下げること
- 対策は具体的かつ実行可能なものにすること
